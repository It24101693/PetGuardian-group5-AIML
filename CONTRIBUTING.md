# Contribution Guidelines

Thank you for your interest in contributing to the PetGuardian project! We appreciate your help in making this project better. Below are the guidelines for contributing:

## How to Contribute

1. **Fork the Repository**: Click on the 'Fork' button in the top right corner of the repository page to create your own copy of the project.

2. **Clone Your Fork**: Clone your forked repository to your local machine using the following command:
   ```bash
   git clone https://github.com/YourUsername/PetGuardian-group5-AIML.git
   ```

3. **Create a New Branch**: Before making any changes, create a new branch for your work:
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make Changes**: Implement your feature or bugfix.

5. **Commit Your Changes**: Commit your changes with a clear message:
   ```bash
   git commit -m "Add your commit message here"
   ```

6. **Push Changes**: Push your changes back to your forked repository:
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**: Go to the original repository where you want to propose your changes. Click on the `Pull Requests` tab, then click on the `New Pull Request` button. Select your branch and submit the pull request.

## Code of Conduct

Please note that this project adheres to a Code of Conduct. By participating, you are expected to uphold this code. For more details, please check the `CODE_OF_CONDUCT.md` file in the repository.

## Issues

If you come across any issues or bugs, please open an issue in the repository with a description of the problem.

Thank you for contributing to PetGuardian! Your efforts help improve the project and serve the community better.
